1348/// Open a new window.
1349///
1350/// Currently this is used to open floating and external windows.
1351/// Floats are windows that are drawn above the split layout, at some anchor
1352/// position in some other window. Floats can be drawn internally or by external
1353/// GUI with the |ui-multigrid| extension. External windows are only supported
1354/// with multigrid GUIs, and are displayed as separate top-level windows.
1355///
1356/// For a general overview of floats, see |api-floatwin|.
1357///
1358/// Exactly one of `external` and `relative` must be specified. The `width` and
1359/// `height` of the new window must be specified.
1360///
1361/// With relative=editor (row=0,col=0) refers to the top-left corner of the
1362/// screen-grid and (row=Lines-1,col=Columns-1) refers to the bottom-right
1363/// corner. Fractional values are allowed, but the builtin implementation
1364/// (used by non-multigrid UIs) will always round down to nearest integer.
1365///
1366/// Out-of-bounds values, and configurations that make the float not fit inside
1367/// the main editor, are allowed. The builtin implementation truncates values
1368/// so floats are fully within the main screen grid. External GUIs
1369/// could let floats hover outside of the main window like a tooltip, but
1370/// this should not be used to specify arbitrary WM screen positions.
1371///
1372/// Example (Lua): window-relative float
1373/// <pre>
1374/// vim.api.nvim_open_win(0, false,
1375/// {relative='win', row=3, col=3, width=12, height=3})
1376/// </pre>
1377///
1378/// Example (Lua): buffer-relative float (travels as buffer is scrolled)
1379/// <pre>
1380/// vim.api.nvim_open_win(0, false,
1381/// {relative='win', width=12, height=3, bufpos={100,10}})
1382/// </pre>
1383///
1384/// @param buffer Buffer to display, or 0 for current buffer
1385/// @param enter Enter the window (make it the current window)
1386/// @param config Map defining the window configuration. Keys:
1387/// - `relative`: Sets the window layout to "floating", placed at (row,col)
1388/// coordinates relative to:
1389/// - "editor" The global editor grid
1390/// - "win" Window given by the `win` field, or current window.
1391/// - "cursor" Cursor position in current window.
1392/// - `win`: |window-ID| for relative="win".
1393/// - `anchor`: Decides which corner of the float to place at (row,col):
1394/// - "NW" northwest (default)
1395/// - "NE" northeast
1396/// - "SW" southwest
1397/// - "SE" southeast
1398/// - `width`: Window width (in character cells). Minimum of 1.
1399/// - `height`: Window height (in character cells). Minimum of 1.
1400/// - `bufpos`: Places float relative to buffer text (only when
1401/// relative="win"). Takes a tuple of zero-indexed [line, column].
1402/// `row` and `col` if given are applied relative to this
1403/// position, else they default to `row=1` and `col=0`
1404/// (thus like a tooltip near the buffer text).
1405/// - `row`: Row position in units of "screen cell height", may be fractional.
1406/// - `col`: Column position in units of "screen cell width", may be
1407/// fractional.
1408/// - `focusable`: Enable focus by user actions (wincmds, mouse events).
1409/// Defaults to true. Non-focusable windows can be entered by
1410/// |nvim_set_current_win()|.
1411/// - `external`: GUI should display the window as an external
1412/// top-level window. Currently accepts no other positioning
1413/// configuration together with this.
1414/// - `style`: Configure the appearance of the window. Currently only takes
1415/// one non-empty value:
1416/// - "minimal" Nvim will display the window with many UI options
1417/// disabled. This is useful when displaying a temporary
1418/// float where the text should not be edited. Disables
1419/// 'number', 'relativenumber', 'cursorline', 'cursorcolumn',
1420/// 'foldcolumn', 'spell' and 'list' options. 'signcolumn'
1421/// is changed to `auto` and 'colorcolumn' is cleared. The
1422/// end-of-buffer region is hidden by setting `eob` flag of
1423/// 'fillchars' to a space char, and clearing the
1424/// |EndOfBuffer| region in 'winhighlight'.
1425/// - `border`: style of (optional) window border. This can either be a string
1426/// or an array. the string values are:
1427/// - "none" No border. This is the default
1428/// - "single" a single line box
1429/// - "double" a double line box
1430/// - "shadow" a drop shadow effect by blending with the background.
1431/// If it is an array it should be an array of eight items or any divisor of
1432/// eight. The array will specifify the eight chars building up the border
1433/// in a clockwise fashion starting with the top-left corner. As, an
1434/// example, the double box style could be specified as:
1435/// [ "", "" ,"", "", "", "", "", "" ]
1436/// if the number of chars are less than eight, they will be repeated. Thus
1437/// an ASCII border could be specified as:
1438/// [ "/", "-", "\\", "|" ]
1439/// or all chars the same as:
1440/// [ "x" ]
1441/// An empty string can be used to turn off a specific border, for instance:
1442/// [ "", "", "", ">", "", "", "", "<" ]
1443/// will only make vertical borders but not horizontal ones.
1444/// By default `FloatBorder` highlight is used which links to `VertSplit`
1445/// when not defined. It could also be specified by character:
1446/// [ {"+", "MyCorner"}, {"x", "MyBorder"} ]
1447///
1448/// @param[out] err Error details, if any

1450/// @return Window handle, or 0 on error
